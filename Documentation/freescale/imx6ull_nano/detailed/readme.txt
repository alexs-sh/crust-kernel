Сборка системы разделена на 3 стадии:
1. подготовка окружения, первоначальная сборка загрузчика, ядра, минимальной КФС (busybox)
2. "тонкая" настройка ядра под конкретную платформу / периферию (опционально)
3. создание корневой файловой системы на базе полноценного дистрибутива Debian/Ubuntu/Arch... (опционально)

На первом этапе выполняется:
    - настройка buildroot
    - получение исходных кодов компиляторов, библиотек, ядра, загрузчика...
    - сборка кросс-компиляторов, библиотек, вспомогательных инструментов
    - сборка загрузчика, ядра, сервисной КФС под целевую платформу
    - создание загрузочного образа для записи на SD-карту

На втором шаге выполняется:
    - добавление/убавление поддерижваемых устройств, файловых систем, сервисов и т.п.
    - изменение конфигураций DeviceTree 
    - сборка ядра и генерация DeviceTreeBlob 

На третьем шаге выполняется:
    - получение бинарных пакетов для устройства из репозитория
    - запуск и первоначальная конфигурация системы
    - создание архива для записи в устройство

Третий шаг может быть выполнен либо на машине разработчика, либо непосредственно на устройсте.
В первом случае необходимо выполнить:
    - установку *bootstrap
    - получение бинарных пакетов из репозитория
    - запуск и конфигурирование системы через эмулятор
    - создание архива с КФС
Во втором случае необходимо выполнить:
    - поиск и загрузку архива КФС / установщика
    - перенос архива на устройство
    - настройку / установку системы
